# Scaleway Deployment

This directory contains the container-native Scaleway deployment for ShelfScan.

## What it creates

- A private Container Registry namespace for the application image.
- A Serverless Containers namespace.
- A managed PostgreSQL instance, database, and application user.
- A private Object Storage bucket for uploaded images.
- An IAM application and API key dedicated to Object Storage access.
- An optional public Serverless Container wired to the generated database and bucket.

## Deployment model

The stateful services are managed by Scaleway. The application itself runs as a stateless container:

- PostgreSQL replaces SQLite.
- Object Storage replaces the local uploads directory.
- The app keeps serving files through its authenticated `/uploads/{key}` route.

## 1. Provision the managed services and registry

Export your Scaleway credentials in the shell where you run OpenTofu:

```bash
export SCW_ACCESS_KEY="..."
export SCW_SECRET_KEY="..."
export SCW_DEFAULT_PROJECT_ID="..."
```

Create the variables file:

```bash
cd deploy/scaleway/terraform
cp terraform.tfvars.example terraform.tfvars
```

For the first apply, keep `deploy_container = false` so OpenTofu creates only the managed services and registry prerequisites:

```bash
tofu init
tofu apply
```

## 2. Build and push the application image

Read the `suggested_registry_image` output, then authenticate Docker to Scaleway Container Registry:

```bash
scw registry login
docker build -t "$(tofu output -raw suggested_registry_image)" .
docker push "$(tofu output -raw suggested_registry_image)"
```

## 3. Deploy the Serverless Container

Set `deploy_container = true` in `terraform.tfvars`.

If you push to a different tag or repository path than the suggested one, also set `container_registry_image` explicitly.

Then apply again:

```bash
tofu apply
```

After that, OpenTofu outputs the generated Scaleway container domain.

## 4. Optional custom domain

If you want a custom hostname:

- Create a DNS `CNAME` pointing to the generated `container_domain_name` output.
- Set `container_hostname` in `terraform.tfvars`.
- Re-run `tofu apply`.

## 5. Verify the deployment

Use the generated public container domain, or your custom hostname if configured:

```bash
curl -fsS "https://$(tofu output -raw container_domain_name)/api/v1/health"
```

You should receive:

```json
{"status":"ok"}
```

## Notes

- The bootstrap admin password is generated by OpenTofu and exposed as a sensitive output.
- The database currently uses a public managed endpoint secured with credentials. The ACL defaults to `0.0.0.0/0` because Serverless Containers do not expose a stable source IP to restrict against here.
- Object Storage remains private. The app accesses it with a dedicated IAM application key.
- If you want always-on warm instances, increase `container_min_scale` from `0`.
